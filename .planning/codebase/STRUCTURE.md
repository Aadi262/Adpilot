## Directory Purposes

**src/config/ — Environment & Singletons:**
- `index.js` — Joi env schema validation (throws on startup if required vars missing); exports all config
- `logger.js` — Winston logger with pino formatting + ALS context injection
- `prisma.js` — Prisma client singleton (re-exported from @prisma/client)
- `redis.js` — ioredis singleton (lazy-initialized on first call)
- `als.js` — AsyncLocalStorage for request context (traceId, teamId, correlationId)
- `sentry.js` — Sentry error tracking initialization
- `featureFlags.js` — Runtime feature toggles (SEO_ENGINE_V2, LIGHTHOUSE_ENABLED, SEO_SUMMARY_ENABLED)
- `seo.js` — SEO audit configuration constants (crawl limits, timeout, lighthouse config)
- `limits.js` — Plan-based resource limits (maxPagesPerAudit, summaryEnabled per tier)

**src/middleware/ — Request/Response Handlers:**
- `auth.js` — JWT verification, role-based access control (RBAC), team ownership guards
- `errorHandler.js` — Normalize Prisma errors, log operational vs unexpected, Sentry capture, response formatting
- `rateLimiter.js` — Global API limiter (120/min), auth limiter (10/15min), heavy limiter (20/min)
- `sanitize.js` — XSS sanitization via `xss` package
- `validate.js` — Joi schema validation decorator factory
- `validateZod.js` — Zod schema validation (returns 422 on failure)
- `correlationId.js` — X-Correlation-Id extraction/generation, ALS store stamping
- `timing.js` — Response time calculation, inject into JSON response

**src/routes/ — HTTP Path Binding:**
- Key files: `authRoutes.js` (login, register, refresh), `campaignRoutes.js` (CRUD), `adRoutes.js` (CRUD + generate), `analyticsRoutes.js` (metrics, overview), `seoRoutes.js` (audit, keyword tracking), `ruleRoutes.js` (rule CRUD), `integrationRoutes.js` (connect, disconnect, refresh), `budgetProtectionRoutes.js` (scan, alerts), `researchRoutes.js` (competitor analysis), `scalingRoutes.js` (readiness prediction)
- Each file: exports Express Router; defines POST/GET/PATCH/DELETE with middleware chain
- Middleware applied per-route: authenticate, requireRole, requireTeamOwnership, validate, rateLimiters

**src/controllers/ — HTTP Handlers:**
- Named by domain (campaign, ad, auth, seo, rule, budget, research, etc.)
- Each controller exports async functions: list, getOne, create, update, remove, plus domain-specific actions
- Pattern: Parse req.body/params/query → call service → catch err for next(err) → return res.json(...)
- Never throws; all errors delegated to middleware error handler

**src/services/ — Business Logic:**
- **High-level services**: authService, campaignService, adService, analyticsService, seoService
- **Specialized services**:
  - `integrations/IntegrationService.js` — OAuth token management; delegates to adapters (Meta, Google, Slack)
  - `integrations/TokenEncryptionService.js` — AES-256-GCM encryption/decryption for stored tokens
  - `rules/RuleEngine.js` — Singleton; evaluates triggered rules; delegates to strategy instances
  - `analytics/MetricsCalculator.js` — Pure functions (ROAS, CPA, CTC, EMA, spend velocity)
  - `analytics/AnomalyDetector.js` — Z-score anomaly detection on metrics
  - `analytics/AnalyticsAggregator.js` — Redis-cached team-level performance overview
  - `seo/audit/AuditOrchestrator.js` — Multi-stage SEO v2 audit pipeline; wall-clock timeout; Puppeteer lifecycle
  - `seo/audit/engines/{CrawlEngine, TechnicalAnalyzer, PerformanceEngine, ScoringEngine}.js` — Focused computation stages
  - `seo/SeoSummaryService.js` — Claude-powered executive summaries (if SEO_SUMMARY_ENABLED)
  - `keywords/SerpService.js` — ValueSERP API client for rank tracking
  - `ai/{GeminiService, AnthropicService, OllamaService, HuggingFaceService, GroqService, CerebraService}` — LLM providers with fallback chain
  - `budgetProtection/BudgetGuardian.js` — Alert rules + fix actions
  - `scaling/ScalingAnalyzer.js` — Campaign readiness scoring algorithm
  - `team/TeamService.js` — Team CRUD, PERMISSIONS matrix, RBAC helpers
  - `team/InviteService.js` — One-time invite token generation, 48h TTL

**src/repositories/ — Prisma Abstraction:**
- `base/BaseRepository.js` — Generic CRUD: findOne, findMany (with pagination), create, update, softDelete, hardDelete, exists, count
- `campaignRepository.js`, `adRepository.js`, `userRepository.js`, `teamRepository.js` — Extend BaseRepository; add model-specific queries
- All queries include soft-delete guard: `deletedAt: null` in WHERE clause
- Pagination: findMany returns `{ items, total }` for cursor/limit pagination

**src/queues/ — Bull Job Queue:**
- `index.js` — Factory: createQueue(name), defines 8 named queues, registers processors, schedules cron jobs
- `processors/*.js` — 8 processor files:
  - `seoAuditProcessor.js` — Calls AuditOrchestrator.run()
  - `seoMonitorProcessor.js` — Checks if monitors are due, enqueues individual audit jobs
  - `keywordSyncProcessor.js` — Syncs keyword ranks via SerpService
  - `ruleEvaluationProcessor.js` — RuleEngine.evaluate() for a campaign
  - `analyticsRefreshProcessor.js` — MetricsCalculator + anomaly detection
  - `integrationSyncProcessor.js` — Fetches performance data from platforms via adapters
  - `notificationsProcessor.js` — Sends in-app/email notifications
  - `tokenHealthCheckProcessor.js` — Daily refresh check for expired OAuth tokens
- All processors wrapped in AsyncLocalStorage context: traceId=jobId, auto-inject into logs

**src/validators/ — Validation Schemas:**
- `authValidator.js` — Joi schemas: login, register, refresh
- `campaignValidator.js`, `adValidator.js`, `ruleValidator.js` — Domain-specific schemas
- Schemas bound to routes via `validate(schema)` middleware decorator

**src/common/ — Shared Utilities:**
- `AppError.js` — Error class with static factory methods (badRequest, unauthorized, forbidden, notFound, conflict, unprocessable, tooManyRequests, serviceUnavailable, internal)
- `pagination.js` — parsePagination(req.query) → { skip, take }
- `response.js` — formatResponse(data) → { success, data, error, meta, responseTime } envelope

**client/src/pages/ — Page Components (28 files):**
- Protected pages (require auth): DashboardPage, CampaignsPage, AnalyticsPage, SeoPage, AdStudioPage, BudgetProtectionPage, CompetitorHijackPage, ScalingPredictorPage, RulesPage, IntegrationsPage, TeamPage, SettingsPage, NotificationsPage, ResearchPage
- Public pages: LandingPage, LoginPage, RegisterPage, PricingPage, DemoLoginPage
- Special: OAuthCallbackPage (integration callback), AcceptInvitePage (email invite), OnboardingPage

**client/src/components/ — Reusable Components:**
- `layout/AppLayout.jsx` — Main wrapper; Sidebar + main content area
- `layout/Sidebar.jsx` — Navigation; "AI Features" section with Budget, Competitor, Scaling shortcuts
- `ui/` — Base components: Button, Input, Modal, Toast, Card, CommandPalette
- `campaigns/` — Campaign card, list, detail views

**client/src/store/ — State Management:**
- `authStore.js` — Zustand: token, user (id, email, name, role, teamId), isDemo, login(), logout(), refreshToken()

**client/src/lib/ — Client Utilities:**
- `api.js` — Axios instance with interceptors: auto-attach token, 401→logout, 500→event, normalize paginated responses
- `exportCsv.js` — CSV export helper
- `exportReport.js` — Report generation helper

**prisma/schema.prisma — Database Schema:**
- 14+ models: User, Team, Campaign, Ad, CampaignAlert, Keyword, SeoAudit, SeoMonitor, Competitor, ResearchReport, Rule, Notification, Integration, ScoringHistory
- All PKs: UUID, auto-generated
- All timestamps: createdAt, updatedAt, deletedAt (nullable for soft-delete)
- Relationships: User→Team (n:1), Campaign→Team (n:1), Ad→Campaign (n:1), Rule→Campaign (n:1), Integration→Team (n:1), etc.
- All columns use snake_case in DB, mapped to camelCase in Prisma client

## Naming Conventions

**Files:**
- Controllers: `*Controller.js` (campaignController, adController, authController)
- Services: `*Service.js` (campaignService, adService, emailService)
- Repositories: `*Repository.js` (campaignRepository, userRepository)
- Routes: `*Routes.js` (campaignRoutes, authRoutes)
- Processors: `*Processor.js` (seoAuditProcessor, keywordSyncProcessor)
- Strategies: `*Strategy.js` (CpaStrategy, RoasStrategy)
- Adapters: `*Adapter.js` (MetaAdapter, GoogleAdapter)
- Engines: `*Engine.js` (CrawlEngine, ScoringEngine)

**Directories:**
- Services by domain: `services/{ai,analytics,integrations,rules,seo,team,keywords}/`
- Nested patterns: `services/seo/audit/{adapters,engines,rules}/`
- Each layer has own dir: `routes/`, `controllers/`, `services/`, `repositories/`, `middleware/`

**Functions:**
- Controllers: verb-noun (list, getOne, create, update, remove) + domain actions (launch, pause, scan, analyze)
- Services: domain-specific (createCampaign, updateBudget, evaluateRules, syncKeywords)
- Repositories: findOne, findById, findMany, create, update, softDelete, hardDelete, exists, count
- Middleware: decorated as (req, res, next) or (req, res, next) → next(err)

**Variables:**
- camelCase: userId, campaignId, teamId, accessToken, expiresAt
- Booleans: isActive, isPaused, isDeleted, hasError
- Collections: campaigns, rules, metrics (plural)
- Config/const: COOLDOWN_MINUTES, DEFAULT_JOB_OPTIONS (SCREAMING_SNAKE_CASE)

**Types/Classes:**
- PascalCase: AppError, BaseRepository, CampaignService, RuleEngine, AuditOrchestrator
- Adapters: *Adapter (MetaAdapter, GoogleAdapter)
- Strategies: *Strategy (CpaStrategy, RoasStrategy)
- Engines: *Engine (CrawlEngine, ScoringEngine)

## Where to Add New Code

**New Feature (Feature Branch):**
- Backend:
  - Add route: `src/routes/featureRoutes.js` (mount in `src/app.js`)
  - Add controller: `src/controllers/featureController.js` (action methods)
  - Add service: `src/services/featureService.js` (business logic)
  - Add repository: `src/repositories/featureRepository.js` (if new model)
  - Add Joi validator: `src/validators/featureValidator.js` (if request body)
  - Update database: `prisma/schema.prisma`, run `npx prisma db push` or `migrate dev`
- Frontend:
  - Add page: `client/src/pages/FeaturePage.jsx` (import App.jsx routes)
  - Add components: `client/src/components/feature/*.jsx` (if complex)
  - Add API calls: inline in page or `client/src/lib/api.js` helper
  - Add store: `client/src/store/featureStore.js` if complex state needed

**New AI Provider (Service Chain Fallback):**
- Add service file: `src/services/ai/NewProviderService.js` (implement interface: generateAds, analyzeCompetitor, etc.)
- Update calling code: try-catch chain in service that uses it (e.g., `src/services/adService.js::generateAdWithAI()`)
- Mark response: `isMock: true` if all providers fail
- Config: Add env var for API key, check in service before instantiating

**New Integration Platform (Adapter Pattern):**
- Add adapter: `src/services/integrations/adapters/NewPlatformAdapter.js` (extend BaseAdapter, implement all methods)
- Update IntegrationService: No change needed (uses adapter registry pattern)
- Add routes: Likely reuses existing `integrationRoutes.js` (already platform-agnostic)
- Update Prisma: Integration model already supports `provider` field; add to seed if needed

**New Rule Trigger Type (Strategy Pattern):**
- Add strategy: `src/services/rules/strategies/NewMetricStrategy.js` (extend BaseStrategy, implement evaluate())
- Register: Add to STRATEGIES map in `src/services/rules/RuleEngine.js` (one-line addition)
- No other changes to RuleEngine or controllers needed

**New Queue Job (Background Task):**
- Add queue: Define in `src/queues/index.js::queues` object
- Add processor: `src/queues/processors/newTaskProcessor.js` (async function wraps business logic)
- Register: Add to `registerProcessors()` function with concurrency level
- Enqueue: Call `queues.newTask.add(data)` from service or controller
- Schedule (if recurring): Add to `scheduleRecurringJobs()` with Bull cron expression

**New Validation Schema:**
- File: `src/validators/featureValidator.js`
- Pattern: Joi object with string/number/array fields, custom().required()
- Mount: Apply via `validate(schema)` middleware in route definition
- Or inline: Zod schema in route handler for query params

**Tests:**
- Unit: `src/**/__tests__/**/*.test.js` or `*.spec.js`
- Framework: Jest (configured in `package.json`)
- Run: `npm test`
- Coverage: `npm test -- --coverage`
